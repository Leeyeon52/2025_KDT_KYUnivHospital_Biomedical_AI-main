# "영상 기반 다중 AI 모델 구강 CDSS"
미래융합교육원에서 이수한 K-디지털 트레이닝 "건양대학교병원 바이오메디컬 인공지능" 과정 최종 프로젝트 결과물입니다.<br/>
과정명에 맞게 의료 관련 데이터를 이용한 프로그램 또는 서비스를 구현하려 하였습니다.<br/>
Image를 이용하여 쉽고, 빠르게 구강 건강 상태를 스마트폰에서 예측할 수 있는 서비스를 만들고자 했습니다.<br/>
개발 기간은 2025년 6월 30일부터 2025년 8월 22일까지 약 2개월 정도였습니다.

### 초기 구현 목표
- 사용자가 스마트폰 또는 현장 촬영 기기로 구강 이미지를 업로드
- 챗봇 AI를 사용, 진단 기록DB에 접근하여 쉽게 이전 기록 등을 알림
- 다중 AI 모델이 영상과 문진 데이터를 통합 분석하여 구강 질환 여부 판단
- 생성형 AI가 분석 및 문진 결과를 자연어로 설명하는 리포트 생성
- 웹/모바일 플랫폼을 통해 사용자에게 진단 결과와 상담 기능 제공
- 필요시 의료진과 비대면 상담으로 연계하여 추가 진료 및 건강관리 지원

### AI 모델
사용한 AI 모델은
일반 사진을 이용하는 Segmentation 치아 번호, 질병, 위생 예측 모델 3가지<br/>
파노라마 X-ray 사진을 이용하는 Classification 임플란트 예측 모델 1가지<br/>
파노라마 X-ray 사진을 이용하는 Detection 충치, 보철물, 근관치료, 상실치아 예측 모델 1가지 입니다.

### Data
데이터는 AI-Hub의 의료데이터를 신청하여 사용했습니다.

### 구조
Backend는 Python Flask를 이용하여 서버를 구축했습니다.<br/>
DB는 MySQL, MongoDB 두 가지를 사용하였고<br/>
그중에서 MySQL에는 일반사용자, 의사 개인정보, 비대면 진료 신청을 저장하고<br/>
MongoDB에는 모델 에측 결과를 저장하였습니다.<br/>
Frontend는 크로스 플랫폼인 Flutter를 이용하여 앱/웹 동시에 개발하였습니다.<br/>

# 아쉬웠던 점
우선 결과적으로 초기에 목표했던 만큼, 완성도 있게 제작하지 못하였습니다.<br/>
최소 기능인 영상을 이용하여 스마트폰으로 구강 상태를 예측, 챗봇으로 DB에 접근하여 이전 기록을 요약 및 설명,<br/>
일반 사용자가 진료 신청을 하여 의사가 comment를 답변 까지는 구현 하였으나 오류, 개선점이 많습니다.<br/>

1. AI 모델 학습에만 급급하여 기존모델과의 비교 테스트를 진행하지 않은점.
2. Firebase나 AWS를 이용 해보지 못한점.
3. Github를 이용한 형상관리 및 협업이 원활하지 못했던 점.
4. 의료 데이터를 사용하는 만큼 데이터 비식별화, DB 보안 쪽에 신경을 쓰지 못한점.
5. 기타 자잘한 오류 등등.(아이디/비밀번호 찾기, 챗봇에서 사진 출력이 원활하지 못함, 일반 사용자 3D 화면, 의사쪽 메인화면)

# 느낀점
- 프로젝트를 할때 Github을 통한 협업, 형상관리에 더 많이 신경을 써야한다
- 서로 개발한 부분을 main에 합칠때는 합치고 나서 테스트가 필요하다
- 초기 기획 당시에 최대한 세세하게 세부사항까지 생각해야 한다(endpoint)
- 개발을 할때는 Minimum Viable Product 부터 시작해서 넓혀나가야 한다
- 중간중간 문서화가 훨씬 중요하다
- 모델 학습시에도 사용한 데이터 비율, 기존 오픈 모델과의 비교 테스트 신경쓰면서 학습해야한다
- 하루, 일주일, 월 단위로 개발구현 목표를 디테일하게 설정하고 개발상황을 공유해야한다
- 팀원간 의사소통이 활발하게 이루어져야 한다
- 개발이 늦어질땐 목표를 수정할줄도 알아야 한다
